# AGENTS.md

# AI-Assisted Development Workflow

## Project

**Movie Ticket Booking System**

A scalable backend application built using Spring Boot that supports multi-city movie ticket booking with seat-level concurrency control, asynchronous notifications, pricing strategies, refund policies, JWT authentication, and AI-powered movie recommendations using Gemini and Pinecone.

---

# AI Agents Used

## 1. OpenAI Codex (Primary Coding Agent)

### Responsibility

Codex was used as the primary implementation assistant inside IntelliJ IDEA.

### Tasks Performed

* Generated project boilerplate
* Generated REST Controllers
* Generated Service implementations
* Generated Repository interfaces
* Generated DTOs
* Generated Entity classes
* Generated Mapper implementations
* Generated Exception handling
* Generated Security configuration
* Generated Scheduler classes
* Generated Event classes
* Generated Notification flow
* Generated Swagger configuration
* Generated Validation annotations

### Human Validation

Every generated class was reviewed before integration.

The following aspects were manually verified:

* Business correctness
* Naming consistency
* Package structure
* Transaction boundaries
* Exception handling
* JPA mappings
* API contracts
* Security configuration

Generated code was modified whenever necessary before being committed.

---

# 3. Gemini API

Gemini was integrated as the application's Large Language Model.

It is used for:

* Movie understanding
* Embedding generation
* Personalized recommendation generation

Gemini is not responsible for application logic.

All business rules remain inside the Spring Boot application.

---

# 4. Pinecone

Pinecone was used as the Vector Database.

Responsibilities:

* Store movie embeddings
* Perform semantic similarity search
* Return relevant movie candidates
* Support Retrieval Augmented Generation (RAG)

# Prompting Strategy

Development was performed using small iterative prompts rather than requesting the complete application in a single step.

Typical workflow:

* Design first
* Review architecture
* Generate one module
* Compile
* Test
* Refactor
* Continue

This approach reduced hallucinations and allowed continuous validation.
